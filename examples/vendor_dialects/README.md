# Vendor dialect integration examples

CompGen can drive end-to-end compilation through third-party MLIR
dialects (NVIDIA CUDA Tile IR, Qualcomm Hexagon MLIR, ...) via a
user-space adapter package. An integration agent — driven by Claude
Code through CompGen's MCP server — explores a vendor repo, produces a
reviewable spec, scaffolds a pip-installable adapter package, and runs
the verification ladder.

## Demo (no GPU, no vendor toolchain)

```bash
.venv/bin/python examples/vendor_dialects/fake_vendor_demo.py
```

Drives the full flow against the fixture at
`tests/fixtures/fake_vendor/`. A regression-proof end-to-end smoke test.

## Driving a real vendor repo via MCP

In Claude Code (or any MCP client):

```
1. scan_vendor_repo(repo_path="/path/to/cuda-tile", target="nvidia-h100",
                    workloads=["tinyllama", "smolvla"])
   → review the returned `descriptor_yaml`
2. scaffold_vendor_package(descriptor_yaml=<edited YAML>,
                           out_dir="/scratch2/agustin/CompGen/tmp/user_perspective_MLIR")
3. pip install -e /scratch2/agustin/CompGen/tmp/user_perspective_MLIR/compgen_cuda_tile
4. verify_vendor_package(package_dir=<path>)
```

Then from Python:

```python
from compgen.api import compile_with_vendor
from compgen.extensions.vendor_dialect import get_adapter

adapter = get_adapter("cuda_tile")
artifact = compile_with_vendor(model, adapter, sample_inputs=(x,))
```

## What lives where

* CompGen core (this repo) — only generic plumbing:
  `python/compgen/extensions/vendor_dialect/`,
  `python/compgen/agent/vendor_integration/`,
  `python/compgen/kernels/providers/claude_kernel.py`,
  `python/compgen/mcp/tools/vendor_dialect.py`.
* User-space (`tmp/user_perspective_MLIR/`) — the scaffolded per-vendor
  packages. Not shipped with CompGen; regenerated by the agent.
